This cookbook walks through a realistic overnight job: you have a CSV of customer reviews and you want a `sentiment` label plus a short list of `topics` for every row. You will submit them as a single batch to the ZeroGPU Batch API, sleep, and wake up to a tagged CSV.

We use [LFM2.5-1.2B-Instruct](/api-reference/models/lfm2-5-1-2b-instruct) because it is small, fast, and good enough for short-form classification on a CPU-cheap budget.

<Note>
**Batch API scope.** The Batch API only accepts `/v1/chat/completions` requests, and every line in a batch must target the same endpoint. Streaming (`"stream": true`) is rejected at create time — batches are inherently non-streaming. If you need a different endpoint or live token streams, use the synchronous APIs instead.
</Note>

<Tip>
**Cheaper model alternatives.** A 1.2B instruct model is overkill for many tagging jobs. If your taxonomy is fixed and short, swap in a classifier head:

- [deberta-v3-small](/api-reference/models/deberta-v3-small) for a sentiment-only label (positive / neutral / negative) at a fraction of the cost.
- [zlm-v1-iab-classify-edge](/api-reference/models/zlm-v1-iab-classify-edge) for topic tagging against the standard IAB content taxonomy when "topics" really means content category.

Both are batch-friendly and noticeably cheaper per row than a chat model. Use LFM only when you genuinely need open-vocabulary topics or a free-form rationale.
</Tip>

<Note>
All sample reviews below are synthetic. Swap them for your own CSV before running in production.
</Note>

| --- | --- | --- |

## Step-by-step flow

The job has five steps. Each one maps to one Batch API call.

| # | Action | API call |
| --- | --- | --- |
| 1 | Build `input.jsonl` from `reviews.csv`, one line per review | local |
| 2 | Upload the JSONL | `POST /v1/files` (`purpose=batch`) |
| 3 | Create the batch | `POST /v1/batches` |
| 4 | Poll until `completed`, `failed`, or `expired` | `GET /v1/batches/{id}` |
| 5 | Download output and error files, merge back into a tagged CSV | `GET /v1/files/{id}/content` |

### The system prompt

Every line in the JSONL is an independent chat completion. We pin the model to JSON-only output so the parser does not have to be clever:

```text
You are a review tagger. For each customer review, respond with a single JSON
object and nothing else, in this exact shape:

{ "sentiment": "...", "topics": ["..."] }

sentiment is one of: "positive", "neutral", "negative".
topics is a list of 1-3 short lowercase strings (for example: "battery",
"shipping", "support", "build quality", "app", "price"). No prose, no code
fences, no extra keys.
```

### A single JSONL line

`custom_id` is **the only way to match a result back to a CSV row**, because the Batch API does not preserve input order in the output file. Use your `review_id` directly.

```json
{
  "custom_id": "r-001",
  "method":    "POST",
  "url":       "/v1/chat/completions",
  "body": {
    "model":       "LFM2.5-1.2B-Instruct",
    "temperature": 0,
    "messages": [
      { "role": "system", "content": "You are a review tagger. ..." },
      { "role": "user",   "content": "Battery lasts two days, love it." }
    ]
  }
}
```

import os, json, requests

BASE    = "https://api.zerogpu.ai"
HEADERS = {
    "x-api-key":    os.environ["ZEROGPU_API_KEY"],
    "x-project-id": os.environ["ZEROGPU_PROJECT_ID"],
}

# 1. Upload
with open("input.jsonl", "rb") as f:
    up = requests.post(
        f"{BASE}/v1/files",
        headers=HEADERS,
        data={"purpose": "batch"},
        files={"file": ("input.jsonl", f, "application/jsonl")},
    ).json()
file_id = up["id"]

# 2. Create
batch = requests.post(
    f"{BASE}/v1/batches",
    headers={**HEADERS, "content-type": "application/json"},
    json={
        "input_file_id":     file_id,
        "endpoint":          "/v1/chat/completions",
        "completion_window": "24h",
        "metadata":          {"job": "review-tagging-2026-06-08"},
    },
).json()
batch_id = batch["id"]
```

</CodeGroup>

<Note>
**Validation runs at create time.** The Batch API parses every line of your JSONL before `POST /v1/batches` returns. A duplicate `custom_id`, a line over 1 MB, or `"stream": true` will reject the whole batch with a 400 that points at the offending `line`. Fix the JSONL locally and resubmit; nothing is charged for a rejected create.
</Note>

## Full Python script

End-to-end: read the CSV, build the JSONL, upload, create, poll, download, merge into `tagged.csv`, and write any failed rows to `failed.csv` for inspection.

```python tag_reviews.py
import csv
import json
import os
import time

import requests

BASE    = "https://api.zerogpu.ai"
HEADERS = {
    "x-api-key":    os.environ["ZEROGPU_API_KEY"],
    "x-project-id": os.environ["ZEROGPU_PROJECT_ID"],
}
MODEL = "LFM2.5-1.2B-Instruct"

SYSTEM_PROMPT = (
    "You are a review tagger. For each customer review, respond with a "
    "single JSON object and nothing else, in this exact shape:\n"
    '{ "sentiment": "...", "topics": ["..."] }\n'
    'sentiment is one of: "positive", "neutral", "negative". '
    "topics is a list of 1-3 short lowercase strings (for example: "
    '"battery", "shipping", "support", "build quality", "app", "price"). '
    "No prose, no code fences, no extra keys."
)


def build_jsonl(csv_path: str, jsonl_path: str) -> int:
    """Skip rows with an empty review; return the number of lines written."""
    written = 0
    with open(csv_path, encoding="utf-8") as src, \
         open(jsonl_path, "w", encoding="utf-8") as dst:
        reader = csv.DictReader(src)
        for row in reader:
            review = (row.get("review") or "").strip()
            if not review:
                # Row 7 in the sample CSV has no review text. Don't waste a
                # request on it; record it as a local skip instead.
                print(f"skip {row.get('review_id')}: empty review text")
                continue
            line = {
                "custom_id": row["review_id"],
                "method":    "POST",
                "url":       "/v1/chat/completions",
                "body": {
                    "model":       MODEL,
                    "temperature": 0,
                    "messages": [
                        {"role": "system", "content": SYSTEM_PROMPT},
                        {"role": "user",   "content": review},
                    ],
                },
            }
            dst.write(json.dumps(line) + "\n")
            written += 1
    return written


def upload(jsonl_path: str) -> str:
    with open(jsonl_path, "rb") as f:
        resp = requests.post(
            f"{BASE}/v1/files",
            headers=HEADERS,
            data={"purpose": "batch"},
            files={"file": (os.path.basename(jsonl_path), f, "application/jsonl")},
        )
    resp.raise_for_status()
    return resp.json()["id"]


def create_batch(file_id: str) -> dict:
    resp = requests.post(
        f"{BASE}/v1/batches",
        headers={**HEADERS, "content-type": "application/json"},
        json={
            "input_file_id":     file_id,
            "endpoint":          "/v1/chat/completions",
            "completion_window": "24h",
            "metadata":          {"job": "review-tagging"},
        },
    )
    resp.raise_for_status()
    return resp.json()


def poll(batch_id: str, interval: int = 30) -> dict:
    while True:
        resp = requests.get(f"{BASE}/v1/batches/{batch_id}", headers=HEADERS)
        resp.raise_for_status()
        batch = resp.json()
        counts = batch.get("request_counts", {})
        print(
            f"{batch['status']}: "
            f"{counts.get('completed', 0)}/{counts.get('total', 0)} done, "
            f"{counts.get('failed', 0)} failed"
        )
        if batch["status"] in ("completed", "failed", "expired", "cancelled"):
            return batch
        time.sleep(interval)


def download(file_id: str) -> str:
    resp = requests.get(f"{BASE}/v1/files/{file_id}/content", headers=HEADERS)
    resp.raise_for_status()
    return resp.text


def parse_tag(content: str) -> dict | None:
    """Model is asked for JSON only, but defensively strip code fences."""
    text = content.strip()
    if text.startswith("```"):
        text = text.strip("`")
        if text.startswith("json"):
            text = text[4:]
    try:
        obj = json.loads(text)
    except json.JSONDecodeError:
        return None
    if not isinstance(obj, dict):
        return None
    sentiment = obj.get("sentiment")
    topics    = obj.get("topics")
    if sentiment not in ("positive", "neutral", "negative"):
        return None
    if not isinstance(topics, list):
        return None
    return {"sentiment": sentiment, "topics": [str(t) for t in topics[:3]]}


def main() -> None:
    n = build_jsonl("reviews.csv", "input.jsonl")
    print(f"wrote {n} requests to input.jsonl")

    file_id = upload("input.jsonl")
    print(f"uploaded {file_id}")

    batch = create_batch(file_id)
    print(f"created {batch['id']}")

    batch = poll(batch["id"])
    if batch["status"] != "completed":
        raise SystemExit(f"batch ended with status {batch['status']}")

    # Output order is NOT input order. Key everything by custom_id.
    tagged: dict[str, dict] = {}
    if batch.get("output_file_id"):
        for line in download(batch["output_file_id"]).splitlines():
            if not line.strip():
                continue
            rec = json.loads(line)
            # Batch API returns OpenAI-style responses for /v1/chat/completions
            content = rec["response"]["body"]["choices"][0]["message"]["content"]
            tag = parse_tag(content)
            if tag is not None:
                tagged[rec["custom_id"]] = tag

    failures: list[dict] = []
    if batch.get("error_file_id"):
        for line in download(batch["error_file_id"]).splitlines():
            if not line.strip():
                continue
            rec = json.loads(line)
            failures.append({
                "review_id": rec["custom_id"],
                "code":      (rec.get("error") or {}).get("code", ""),
                "message":   (rec.get("error") or {}).get("message", ""),
            })

    # Merge: walk the CSV in its original order, attach tags by custom_id.
    with open("reviews.csv",  encoding="utf-8") as src, \
         open("tagged.csv", "w", encoding="utf-8", newline="") as dst:
        reader = csv.DictReader(src)
        writer = csv.writer(dst)
        writer.writerow(["review_id", "review", "sentiment", "topics"])
        for row in reader:
            tag = tagged.get(row["review_id"], {})
            writer.writerow([
                row["review_id"],
                row.get("review", ""),
                tag.get("sentiment", ""),
                "|".join(tag.get("topics", [])),
            ])

    if failures:
        with open("failed.csv", "w", encoding="utf-8", newline="") as f:
            writer = csv.DictWriter(f, fieldnames=["review_id", "code", "message"])
            writer.writeheader()
            writer.writerows(failures)
        print(f"{len(failures)} rows failed, see failed.csv")

    print(f"done. tagged {len(tagged)}/{n} reviews -> tagged.csv")


if __name__ == "__main__":
    main()
```

A successful output line looks like this:

```json
{
  "id": "batch_req_a1",
  "custom_id": "r-001",
  "response": {
    "status_code": 200,
    "request_id": "req_xyz",
    "body": {
      "choices": [
        { "message": { "role": "assistant",
                       "content": "{\"sentiment\":\"positive\",\"topics\":[\"battery\"]}" } }
      ]
    }
  }
}
```

A failed line looks like this (`response` is `null`, `error` is populated):

```json
{
  "id": "batch_req_a7",
  "custom_id": "r-007",
  "response": null,
  "error": {
    "code": "invalid_request_error",
    "message": "messages[1].content must be a non-empty string",
    "param": "messages[1].content"
  }
}
```

<Warning>
**Output order is not preserved.** Lines in `output.jsonl` (and `error.jsonl`) can arrive in any order. Always key by `custom_id`. If two of your rows share the same `custom_id`, the batch is rejected at create time, so pick a column that is already unique (here, `review_id`).
</Warning>

## Error recovery

A batch ends in one of four terminal states, and only `completed` guarantees an `output_file_id`. Even a `completed` batch can have a populated `error_file_id` when *some* lines failed but the batch as a whole ran.

| Status | What it means | What to do |
| --- | --- | --- |
| `completed` | Batch ran to the end. Some lines may still have failed. | Read `output_file_id` for successes, `error_file_id` for per-line failures. |
| `failed` | The batch itself never made progress (auth, quota, infra). | Inspect the top-level `errors` array, fix, resubmit the same input file. |
| `expired` | Did not finish inside the 24h `completion_window`. | Same as `failed`, plus consider splitting the input into smaller batches. |
| `cancelled` | You called `POST /v1/batches/{id}/cancel`. | Resubmit when ready. |

### Resubmit only the failed lines

For the common case (`completed` with a non-empty `error_file_id`), do not re-run the whole job. Build a follow-up JSONL from the original `input.jsonl` keyed by the `custom_id`s that appear in the error file:

```python recover.py
import json
import os

import requests

BASE    = "https://api.zerogpu.ai"
HEADERS = {
    "x-api-key":    os.environ["ZEROGPU_API_KEY"],
    "x-project-id": os.environ["ZEROGPU_PROJECT_ID"],
}

def errored_ids(error_file_id: str) -> set[str]:
    text = requests.get(
        f"{BASE}/v1/files/{error_file_id}/content", headers=HEADERS
    ).text
    return {
        json.loads(line)["custom_id"]
        for line in text.splitlines() if line.strip()
    }

def retry(input_jsonl: str, error_file_id: str) -> str:
    ids = errored_ids(error_file_id)
    with open(input_jsonl, encoding="utf-8") as src, \
         open("retry.jsonl", "w", encoding="utf-8") as dst:
        for line in src:
            rec = json.loads(line)
            if rec["custom_id"] in ids:
                dst.write(line)
    # ... upload retry.jsonl and create a new batch the same way as before.
    return "retry.jsonl"
```

For the malformed row in the sample CSV (`r-007`), the right fix is **upstream** — drop or backfill the row before building the JSONL. The local `build_jsonl` in the main script already does that, which is why `r-007` never reaches the API.

### Common per-line errors

| `error.code` | Likely cause | Fix |
| --- | --- | --- |
| `invalid_request_error` | Empty content, bad role, unsupported field. | Repair the offending line locally and retry. |
| `rate_limit_exceeded` | Project quota tripped mid-batch. | Wait, raise quota, then retry the failed `custom_id`s. |
| `model_error` | Transient model failure on that line. | Safe to retry as-is. |
| `timeout` | Single line took too long. | Shorten the input, lower `max_tokens`, then retry. |

See the [Errors reference](/docs/batch/errors) for the full table.

## Other Batch API endpoints

The four endpoints used above (`POST /v1/files`, `POST /v1/batches`, `GET /v1/batches/{id}`, `GET /v1/files/{id}/content`) drive the happy path. A handful of housekeeping endpoints round out the surface — useful for inspection, cleanup, and aborting a stuck job.

| Endpoint | Use it to |
| --- | --- |
| `GET /v1/files` | List uploaded files for the project, newest first. Pass `purpose=batch` or `purpose=batch_output` to scope. Page with `after=<last_id>`. |
| `GET /v1/batches` | List batches for the project, newest first. Cheap way to find an in-flight or recent batch when you've lost the ID. |
| `DELETE /v1/files/{id}` | Delete an input, output, or error file before its `expires_at`. Useful for cost cleanup or compliance after you've persisted results elsewhere. |
| `POST /v1/batches/{id}/cancel` | Cancel an in-flight batch. The status moves to `cancelling`, then `cancelled`; any lines already completed remain in `output_file_id`. |

## Production tips

- **One CSV column for `custom_id`.** Pick a column that is already unique (an order ID, a UUID, a review hash). Duplicates are rejected at create time, and they break your join even if they slipped through.
- **Pin `temperature: 0`.** For classification you want determinism. It also makes failed-line retries idempotent.
- **Trust the schema, but verify.** Even with a strict system prompt, models occasionally emit a code fence or a stray sentence. Keep a `parse_tag` step and treat parse failures as their own bucket — do not silently drop them into `sentiment=""`.
- **Cap batch size by your real SLO, not the 100 MB file limit.** A million-line batch that expires after 24h is worse than ten 100k-line batches that finish in two hours each. Pick a chunk size that comfortably fits the `completion_window`.
- **Log `request_counts` on every poll.** It is the cheapest progress signal you have, and a stalled `completed` counter is usually the first sign something is wrong upstream of the model.
- **Keep the input JSONL around** until you have read both `output_file_id` and `error_file_id`. Files expire (`expires_at` on the file object), and recovery needs the original line bodies, not just the IDs.
- **Don't poll faster than 30s.** It costs you nothing to wait, and the batch status only changes on minute-scale transitions.

## Go deeper

If "topics" in your taxonomy really means **content category** (ads, brand safety, recommendation buckets), drop the chat model and use a purpose-built classifier instead. [`zlm-v1-iab-classify-edge`](/api-reference/models/zlm-v1-iab-classify-edge) returns IAB-standard categories directly, runs cheaper than a 1.2B instruct model, and slots into the same Batch JSONL flow — just swap the `body.model` in each line and drop the system prompt. Same `custom_id` join, same `output_file_id` download path.

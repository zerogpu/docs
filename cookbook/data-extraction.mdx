Use extraction models to turn free-form text into **structured JSON** or **labeled spans**. For example: contact blocks from email signatures, technologies from product descriptions, or PII from user-submitted content.

ZeroGPU routes these requests through the [Responses API](/api-reference/endpoint/responses). Pass model-specific options in the **`metadata`** object (`usecase`, `schema`, `labels`, and related fields).

## Structured field extraction

Use **`gliner2-base-v1`** with `metadata.usecase` set to **`json`** and a **`schema`** that describes the fields you want. Each field uses the form `name::type::description`.

### Request

<CodeGroup>

```bash cURL
curl --location 'https://api.zerogpu.ai/v1/responses' \
  --header 'content-type: application/json' \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'x-project-id: YOUR_PROJECT_ID' \
  --data '{
    "model": "gliner2-base-v1",
    "input": "Best regards, John Smith, Senior Software Engineer at Acme Corp. Phone: (555) 123-4567, Email: john.smith@acme.com, Office: 123 Main Street, Suite 400, San Francisco, CA 94105",
    "metadata": {
      "usecase": "json",
      "schema": {
        "contact": [
          "name::str::Full name",
          "title::str::Job title",
          "company::str::Company name",
          "phone::str::Phone number",
          "email::str::Email address",
          "address::str::Office address"
        ]
      }
    }
  }'
```

```python Python
import json
import requests

url = "https://api.zerogpu.ai/v1/responses"
headers = {
    "content-type": "application/json",
    "x-api-key": "YOUR_API_KEY",
    "x-project-id": "YOUR_PROJECT_ID",
}

payload = {
    "model": "gliner2-base-v1",
    "input": (
        "Best regards, John Smith, Senior Software Engineer at Acme Corp. "
        "Phone: (555) 123-4567, Email: john.smith@acme.com, "
        "Office: 123 Main Street, Suite 400, San Francisco, CA 94105"
    ),
    "metadata": {
        "usecase": "json",
        "schema": {
            "contact": [
                "name::str::Full name",
                "title::str::Job title",
                "company::str::Company name",
                "phone::str::Phone number",
                "email::str::Email address",
                "address::str::Office address",
            ]
        },
    },
}

response = requests.post(url, headers=headers, json=payload)
response.raise_for_status()
body = response.json()

# Parse model output from the Responses API envelope
text = body["output"][0]["content"][0]["text"]
extracted = json.loads(text)
print(json.dumps(extracted, indent=2))
```

</CodeGroup>

### Response

The model returns JSON in `output[].content[].text`. The shape follows your schema. For example, a `contact` object with the fields you defined. Parse that string with `json.loads` (Python) or `JSON.parse` (JavaScript) before passing it to your app.

## Named entity extraction

For **open-ended labels** (not a fixed field list), use the same model with `metadata.usecase` set to **`ner`** and a **`labels`** array:

<CodeGroup>

```bash cURL
curl --location 'https://api.zerogpu.ai/v1/responses' \
  --header 'content-type: application/json' \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'x-project-id: YOUR_PROJECT_ID' \
  --data '{
    "model": "gliner2-base-v1",
    "input": "The application is built with Python 3.11 and uses PostgreSQL 15 for storage. It runs on Kubernetes with Docker containers and communicates via gRPC.",
    "metadata": {
      "usecase": "ner",
      "labels": ["programming language", "database", "technology", "protocol"],
      "threshold": 0.3
    }
  }'
```

```python Python
import requests

url = "https://api.zerogpu.ai/v1/responses"
headers = {
    "content-type": "application/json",
    "x-api-key": "YOUR_API_KEY",
    "x-project-id": "YOUR_PROJECT_ID",
}
payload = {
    "model": "gliner2-base-v1",
    "input": (
        "The application is built with Python 3.11 and uses PostgreSQL 15 for storage. "
        "It runs on Kubernetes with Docker containers and communicates via gRPC."
    ),
    "metadata": {
        "usecase": "ner",
        "labels": ["programming language", "database", "technology", "protocol"],
        "threshold": 0.3,
    },
}
response = requests.post(url, headers=headers, json=payload)
print(response.json())
```

</CodeGroup>

The response includes **entity spans** with labels and confidence scores. Adjust `threshold` to trade off precision and recall.

## PII extraction

For **personally identifiable information**, use **`gliner-multi-pii-v1`** with `metadata.usecase` set to **`extract-pii`**:

<CodeGroup>

```bash cURL
curl --location 'https://api.zerogpu.ai/v1/responses' \
  --header 'content-type: application/json' \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'x-project-id: YOUR_PROJECT_ID' \
  --data '{
    "model": "gliner-multi-pii-v1",
    "input": "Contact John Doe at john@example.com or +1-415-555-0134.",
    "metadata": {
      "usecase": "extract-pii",
      "threshold": 0.5,
      "categories": ["identity", "contact"]
    }
  }'
```

```python Python
import requests

url = "https://api.zerogpu.ai/v1/responses"
headers = {
    "content-type": "application/json",
    "x-api-key": "YOUR_API_KEY",
    "x-project-id": "YOUR_PROJECT_ID",
}
payload = {
    "model": "gliner-multi-pii-v1",
    "input": "Contact John Doe at john@example.com or +1-415-555-0134.",
    "metadata": {
        "usecase": "extract-pii",
        "threshold": 0.5,
        "categories": ["identity", "contact"],
    },
}
response = requests.post(url, headers=headers, json=payload)
print(response.json())
```

</CodeGroup>

To **redact** PII instead of returning spans, use `metadata.usecase` **`redact`** with `mask` set to `label` or another supported mask mode. See the [gliner-multi-pii-v1](/models/gliner-multi-pii-v1) model page for use-case details.

## Tips

- **`input` shape:** GLiNER models accept a **plain string** for `input` (as above). Other models may expect a message array. See [Responses](/api-reference/endpoint/responses#input-shape).
- **Schema design:** Use clear field descriptions in `name::type::description`; they guide what the model extracts.
- **Batch workloads:** Extract many documents with [batch patterns](/cookbook/batch-requests) (parallel requests, per-item error handling).
- **Try interactively:** Open [gliner2-base-v1](/models/gliner2-base-v1) or [gliner-multi-pii-v1](/models/gliner-multi-pii-v1) in the model catalog and use the playground to switch use cases and edit the request body.
- **Full walkthrough:** See the [resume & profile extraction tutorial](/cookbook/data-extraction-tutorial) with a [synthetic dataset](https://github.com/zerogpu/docs/tree/main/tutorials/data-extraction/dataset) and batch script.
